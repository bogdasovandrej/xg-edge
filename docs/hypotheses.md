# Реестр фальсифицируемых гипотез

Гипотеза формулируется до прогона и обязана иметь порог провала. Версия 0.2
разделяет два периода:

- development: матчи до 2025-07-01; тестовые walk-forward окна начинаются
  2023-07-01, n = 745;
- locked retrospective holdout: сезон 2025/26, n = 375 пригодных прогнозов.

Development используется для допуска признаков. Holdout — только для итоговой
оценки зафиксированного контура. Следующий необходимый этап — полностью
проспективный сезон 2026/27.

Числа абляций воспроизводятся scripts/run_hypotheses.py и сохраняются в
reports/hypotheses.csv.

## Проверено на development

Порог поддержки: удаление механизма или замена конфигурации должна ухудшать
log-loss минимум на 0.002. Кандидат не допускается, если не улучшает BASE на
тот же порог.

| ID | Проверка | Log-loss | Δ к BASE | Вердикт |
| --- | --- | ---: | ---: | --- |
| BASE | raw xG, no opponent adjustment, decay 180d | 0.9554 | 0.0000 | зафиксирована |
| H2 | добавить opponent-strength normalization | 0.9571 | +0.0018 | не допущена: улучшения нет |
| H7 | заменить raw xG на npxG | 0.9568 | +0.0014 | не допущена: улучшения нет |
| H8 | убрать time decay | 0.9705 | +0.0151 | подтверждена |
| H9 | зафиксировать rho = 0 | 0.9549 | −0.0005 | не подтверждена для 1X2 |
| — | half-life 90d вместо 180d | 0.9577 | +0.0024 | хуже |
| — | half-life 365d вместо 180d | 0.9596 | +0.0042 | хуже |
| — | GBM challenger против GLM | 0.9675 против 0.9554 | +0.0121 | challenger отклонён |

H2 и H7 не названы доказанно вредными: их ухудшение меньше порога 0.002 по
отдельности. Они исключены из BASE по правилу сложности — недоказанный признак
не получает место в модели.

Поправка Dixon–Coles сохранена в score matrix для exact-score распределения,
но H9 показывает, что она не улучшает агрегированный 1X2 на development.

## Правила допуска

1. Формулировка, метрика и порог фиксируются до эксперимента.
2. Только хронологический walk-forward; случайный split запрещён.
3. Подбор выполняется на development, финальная оценка — на отдельном holdout.
4. Одновременно меняется одна гипотеза; взаимодействия проверяются отдельно.
5. Недоказанный кандидат не входит в BASE.
6. Провал и неопределённость документируются, а не удаляются.
7. После просмотра holdout модель не перенастраивается на нём.
8. Каждые полгода — ревизия; следующий настоящий тест — live 2026/27.
