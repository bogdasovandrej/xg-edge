# Реестр фальсифицируемых гипотез

Формат: **утверждение → метрика → порог провала**. Право упасть есть у каждой
гипотезы; гипотеза, не способная провалиться, не несёт информации. Гипотеза не
подтверждается на данных, где она придумана, — только out-of-sample
(walk-forward, тестовый период 2023/24–2025/26, n = 1120).

Провалившаяся гипотеза документируется здесь с вердиктом — она не удаляется
молча и не «дожимается» переподбором окна. Числа — из
[reports/hypotheses.csv](../reports/hypotheses.csv), протокол — 
`scripts/run_hypotheses.py`.

## Статус: проверенные (v0.1, прогон 2026-07)

| ID | Утверждение | Критерий провала | Результат | Вердикт |

## Статус: ожидают проверки

Блок A — накопленные паттерны (проверяются заново на out-of-sample):

| ID | Утверждение | Критерий провала |
|---|---|---|

| H3 | Удаления через xG-мультипликатор лучше выбрасывания матчей | выбрасывание калибрует не хуже (сейчас реализован пониженный вес матча с красной; минутные мультипликаторы требуют event-данных) |
| H4 | Dead-rubber матчи непредсказуемы — флаг работает на исключение | дисперсия ошибки на них не выше обычной |
| H5 | «Хватает ничьей → безопасный 1X» — ловушка | устойчивый ROI > 0 таких ставок опровергает ловушку |
| H6 | Clean-sheet-ноги фаворитов теряют эдж к плей-офф | эдж равный |

## Протокол допуска

1. Формулировка **до** прогона: «добавление X улучшает log-loss/Brier на
   walk-forward минимум на Δ = 0.002» + порог провала.
2. A/B: модель с механизмом против модели без него, идентичный протокол,
   данные вне периода разработки.
3. Устойчивость по сезонам, а не среднее за счёт одного окна.
4. Бюджет сложности: ≤ 8–12 активных признаков в GLM на ~380 матчей сезона,
   регуляризация обязательна.
5. Вердикт заносится сюда; ревизия реестра — раз в полгода.
